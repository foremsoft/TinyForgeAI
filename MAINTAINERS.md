# Maintainers

This document lists the maintainers of the TinyForgeAI project.

## Core Maintainers

Core maintainers have full commit access and are responsible for:
- Reviewing and merging pull requests
- Triaging issues
- Making release decisions
- Setting project direction

## Becoming a Maintainer

Maintainers are added based on:
- Consistent, high-quality contributions
- Understanding of project architecture
- Positive community interactions
- Commitment to the project's long-term success

If you're interested in becoming a maintainer, start by:
1. Contributing code and documentation
2. Helping with issue triage
3. Reviewing pull requests
4. Participating in discussions

## Maintainer Responsibilities

### Code Review
- Review PRs within 1 week when possible
- Provide constructive feedback
- Ensure code quality and test coverage
- Check for security issues

### Issue Triage
- Label issues appropriately
- Close duplicates and invalid issues
- Ask for clarification when needed
- Prioritize based on impact

### Releases
- Follow the release process in [docs/release.md](docs/release.md)
- Write release notes
- Ensure all tests pass before release
- Announce releases appropriately

### Community
- Respond to questions respectfully
- Welcome new contributors
- Enforce code of conduct
- Foster inclusive environment

## Decision Making

Major decisions are made through:
1. Discussion in GitHub issues or discussions
2. Consensus among maintainers
3. Documentation of decisions

For breaking changes or major features:
- Open an issue for discussion first
- Allow at least 1 week for community feedback
- Document the decision in the relevant PR

## Contact

For maintainer-related questions:
- Open a GitHub issue
- Tag maintainers in discussions
